ndnQoS documentation (NMSU extension of ndnSIM v2.7)

We implement a Quality of Service (QoS) aware network architecture that aims to satisfy low latency, high bandwidth, and high reliability requirements of communications by extending the features of NDN. In this architecture, we categorize traffic into three priority classes (high, medium, and low priority) to enable preferential treatment of traffic flows. The three classes were identified for a smart grid application, but can be generalized to increase or decrease the number of classes. For the smart grid application, the three classes correspond to: (i) protection traffic, which requires low latency and high reliability; (ii) control traffic, which needs high reliability; and (iii) best-effort traffic, which has no requirement constraints.
